API Basejump: Timestamp microservice

User stories:
    1) I can pass a string as a parameter, and it will check to see whether that string contains either a unix timestamp or a natural language date (example: January 1, 2016)
    2) If it does, it returns both the Unix timestamp and the natural language form of that date.
    3) If it does not contain a date or Unix timestamp, it returns null for those properties.

Example usage:

https://fcc-bucky-timestamp.herokuapp.com/November%2011,%202008
https://fcc-bucky-timestamp.herokuapp.com/1226390400

Example output:

{ "unix": 1226390400, "natural": "November 11, 2008" }

Implementation note:

Dates such as February 31, 2002 will return
{"unix":1015113600,"natural":"March 3, 2002"}

Very little data validation has been implemented. This is the same behavior as implemented in the example app.
